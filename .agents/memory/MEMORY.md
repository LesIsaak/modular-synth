- [Distortion CV pattern](distortion-cv-pattern.md) — WaveShaper.curve is not an AudioParam; use AnalyserNode polling (setInterval 32ms) for drive/fold/bits/factor CV inputs.

- [Patch example name & port alignment](patch-example-alignment.md) — Build-in-rack uses PATCH_MODULE_ALIASES in synth-app.tsx; Output ports are 'IN L'/'IN R'; Clock Gen has no CLK out (use GATE); KNIGHT GATE has no CLK in (use SYNC); Drum Machine uses K-TRG/S-TRG/HC-T inputs and K-OUT output.
- [Param change performance pattern](param-change-perf.md) — Audio update is immediate (direct setParam call); React state update wraps setModules in startTransition; handleMidiMon must use modulesRef/focusedModuleIdRef (not captured state) so its deps array stays stable.
- [Pitch shift delay-mod bugs](pitch-shift-delay-bugs.md) — Two critical bugs: (1) DC doubling: setting both delayTime.value=DC_OFF AND connecting a ConstantSource at DC_OFF doubles the centre delay — always zero the AudioParam nominal if using a ConstantSource for DC. (2) Wrong lfoFreq formula: pitch UP needs (2^(n/12)−1)/(2*DELAY_AMT), pitch DOWN needs (1−2^(−n/12))/(2*DELAY_AMT); the two are NOT interchangeable for large shifts.
- [Cable rendering & drag perf](cable-render-perf.md) — pending cable drawn imperatively via refs (no per-RAF setState, starves 8ms-lead MIDI scheduler); PatchCables memo-buster MUST be cableLayoutVersion (post-commit), not modules.
- [ModulePanel memo & prop stability](modulepanel-memo-stability.md) — ModulePanel is React.memo'd; every prop at its call site must be referentially stable or per-RAF drag re-renders all panels and jitters main-thread audio. Use ref-trampoline handlers; memoize derived props on moduleTypeSig not module x/y.
- [poly_step lookahead gate-off bug](poly-step-lookahead-gateoff.md) — gate-off setTimeout must add audioLeadMs offset; swing setTimeout must capture+restore _currentTickAudioTime; knob startTransition defers React renders away from tick messages.
- [Chrome silence-propagation pruning](chrome-silence-pruning.md) — Chrome marks any GainNode with gain=0 (or a path provably producing zeros) as silent and skips its entire upstream subgraph. Affects: (1) zero-gain sinks for AnalyserNode activation — gain must be ≥0.001; (2) GainNode whose AudioParam is driven by audio-rate connection — if nominal gain=0 and Chrome prunes before evaluating the connection, the source node never runs. Fix: use gain=0.001 (not 0) anywhere a non-silent branch is required.
- [BiquadFilter sub-10Hz instability](biquad-lowfreq.md) — BiquadFilter LP at <10 Hz has coefficient 1−cos(ω₀)≈ω₀²/2 which underflows in float32 (and is marginal in float64). Do NOT use BiquadFilter as a slow envelope smoother (release times >100ms need LP <10Hz). Use JS polling with exp() coefficient instead.
- [OscillatorNode phase offset via PeriodicWave](oscillator-phase-periodicwave.md) — Chrome ignores past start timestamps; oscillator.start(now - T/2) gives phase 0, not 180°. Bake phase into Fourier coefficients: sawtooth at phase π has imag[n] = (-1)^(n+1) × 2/(nπ); cosine uses real[1]=1. All oscillators then start at ctx.currentTime.
